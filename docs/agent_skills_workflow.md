# 🤖 Agent Skills Workflow Guide

คู่มือนี้อธิบายวิธีการเรียกใช้งานและลำดับการทำงาน (Workflow) ของ Custom Skills ทั้ง 4 ตัว (`grill-with-docs`, `karpathy-guidelines`, `scrutinize`, `handoff`) เพื่อให้ได้ผลลัพธ์ระดับ Senior Engineer ที่มีมาตรฐานและลดข้อผิดพลาด

---

## 🔁 ลำดับการใช้งาน (The Senior Workflow)

ถ้าคุณอยากทำงานให้เหมือนวิศวกรซอฟต์แวร์ระดับท็อป นี่คือลำดับการใช้งานสกิลที่คุณ **ต้อง** ทำตาม:

### 1. 🏗️ `/grill-with-docs` (ช่วงออกแบบ - Design & Plan)
**เมื่อไหร่ควรใช้:** ตอนที่คุณกำลังจะเริ่มทำฟีเจอร์ใหม่ หรือสถาปัตยกรรมใหม่ (ห้ามเขียนโค้ดก่อนเด็ดขาด!)
**ทำหน้าที่อะไร:** AI จะสัมภาษณ์และไล่บี้คุณเรื่อง Design, ตั้งคำถามถึง Edge Cases, และเขียนเอกสาร ADR (Architecture Decision Record) ให้คุณโดยอัตโนมัติ เพื่อให้แผนงานชัดเจนก่อนลงมือ
**ตัวอย่างการเรียกใช้:**
> "อยากทำระบบอัปโหลดไฟล์ /grill-with-docs ช่วยรีวิวแผนและทำเอกสารให้หน่อย"

### 2. 💻 `/karpathy-guidelines` (ช่วงเขียนโค้ด - Implementation)
**เมื่อไหร่ควรใช้:** ตอนที่แผนงานนิ่งแล้ว และถึงเวลาให้ AI เริ่มเขียนหรือ Refactor โค้ด
**ทำหน้าที่อะไร:** บังคับให้ AI เขียนโค้ดแบบรัดกุมที่สุด (Simplicity First) ไม่คิดเองเออเอง ไม่สร้าง Abstraction ล่วงหน้าเผื่ออนาคตที่ไม่มีวันมาถึง และเน้นโค้ดที่อ่านง่ายตรวจสอบได้
**ตัวอย่างการเรียกใช้:**
> "เริ่มเขียนฟีเจอร์อัปโหลดไฟล์ตาม ADR ที่คุยกันไว้ และใช้ /karpathy-guidelines คุมกฎการเขียนโค้ดด้วย"

### 3. 🔍 `/scrutinize` (ช่วงตรวจสอบ - Code Review)
**เมื่อไหร่ควรใช้:** เมื่อเขียนโค้ดเสร็จแล้ว (ก่อนจะ Commit หรือ Deploy)
**ทำหน้าที่อะไร:** สวมบทบาทเป็นคนนอกที่มาไล่ Review โค้ดของคุณแบบเจาะลึก (End-to-End) ทะลุ Call Graph ไม่ใช่แค่ดู Diff โง่ๆ แต่ตั้งคำถามว่ามันทำงานได้จริงตามเป้าไหม มีวิธีที่ง่ายกว่านี้ไหม
**ตัวอย่างการเรียกใช้:**
> "ผมเขียน useAuth เสร็จแล้ว ช่วย /scrutinize ไฟล์นี้เทียบกับ `layout.tsx` ให้หน่อยว่าพังตรงไหนบ้าง"

### 4. 🤝 `/handoff` (ช่วงส่งไม้ต่อ - Context Switch)
**เมื่อไหร่ควรใช้:** เมื่อหมดวัน, งานยาวเกินไปจน Context ของ AI เริ่มล้น, หรือต้องการเปลี่ยนไปทำเรื่องอื่นชั่วคราว
**ทำหน้าที่อะไร:** สรุปสิ่งที่คุยกันทั้งหมด ปัญหาที่เจอ และสิ่งที่จะทำต่อไป ออกมาเป็นไฟล์ Handoff Document เพื่อให้ Agent ตัวใหม่สามารถมารับช่วงต่อได้ทันทีโดยไม่ต้องอธิบายใหม่ตั้งแต่ศูนย์
**ตัวอย่างการเรียกใช้:**
> "วันนี้พอแค่นี้ก่อน ช่วยทำ /handoff ให้หน่อย พรุ่งนี้ค่อยมาลุยเรื่องเชื่อม API ต่อ"

---

## 🎯 สรุป Workflow ใน 1 ประโยค
**"โดนไล่บี้ตอนออกแบบ (`grill-with-docs`) -> เขียนโค้ดแบบเรียบง่ายแต่รัดกุม (`karpathy-guidelines`) -> รีวิวโค้ดหาจุดตาย (`scrutinize`) -> แล้วสรุปงานก่อนเลิก (`handoff`)"**

> **ข้อควรระวัง:** การข้ามขั้นตอนไปให้ AI ปั่นโค้ดมั่วๆ โดยไม่ผ่านการออกแบบ (Grill) หรือไม่ตรวจสอบ (Scrutinize) คือหายนะของโปรเจค! กรุณาทำตามลำดับนี้ถ้าไม่อยากมานั่งแก้ Technical Debt ในอนาคต
